less

  • css 预处理器 less官网 ，bootstrap中less教程：bootstrap
  • Less 是一门 CSS 预处理语言，它扩展了 CSS 语言，增加了变量、Mixin、函数等特性，使 CSS 更易维护和扩展。
  • LESS 既可以在 客户端 上运行 ，也可以借助Node.js在服务端运行。
  • Less编译工具 koala 官网

用法

安装

  • 服务器上安装Less最简单的方法是通过npm, node.js包管理器

命令行用法

  • 装完成后,可以从命令行调用编译器
  • 这将把编译后的CSS输出到标准输出。将CSS结果保存到您所选择的文件中
  • 要输出缩小的CSS,你可以使用clean-css插件 安装插件后,使用 clean-css选项指定一个缩小的CSS输出

代码用法

  • 可以调用编译器的节点

  • 通过一些选项来编译器 （配置）

浏览器端用法

使用更少。 js在浏览器中发展很好,但不适合生产

  • 客户端是最简单的方法开始和发展较低,但在生产中,当性能和可靠性是非常重要的, 我们建议使用节点预编译。 js或第三方工具之一 。
  • 选项是由设置在全局 less 对象 之前的 script

使用

less中的注释

   以//开头的注释，不会被编译到css文件中
   以/**/包裹的注释会被编译到css文件中

less中的变量

变量是块级作用域

  • 使用@来申明一个变量：@color：pink;
  1. 作为普通属性值来使用：直接使用@pink
  2. 作为选择器和属性名：#@{selector的值}的形式
  3. 作为URL：@{url}
  4. 变量的延迟加载

  2. & 的使用

less中的混合

混合就是将一系列属性从一个规则集引入到另一个规则集的方式
规定使用 .混合名 带点

  1. 普通混合
  2. 不带输出的混合
  3. 带参数的混合
  4. 带参数并且有默认值的混合
  5. 带多个参数的混合
  6. 命名参数
  7. 匹配模式
  8. arguments变量

less运算

  • 在less中可以进行加减乘除的运算
  • 计算的一方带单位就行

less避免编译

  • 使用 ~ “” less内容会直接编译成 ~ 内的内容

less继承

  • 性能比混合高
  • 灵活度比混合低
  • less继承方便代码模块化
  • 继承不支持带参数
  • 如社区公共头部与尾部

    使用 extend

  • 语法：获得继承名：extend（继承部分名）{…}

    继承所有状态（如伪类选择器）

  • 语法：获得继承名：extend（继承部分名 all）{…} */
